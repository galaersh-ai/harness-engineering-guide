# Сравнение основных реализаций харнесов

Это фактическое сравнение наиболее известных харнесов для агентов по состоянию на середину 2025 года. Область развивается быстро — детали могут меняться, но архитектурные паттерны остаются стабильными.

## Подробный разбор: ключевые отличия

### Стратегия контекста

Как каждый харнес решает, что показать модели:

```
Aider          : Explicit — user adds files manually (/add, /drop)
Claude Code    : On-demand — agent reads files as needed
Cursor         : Semantic — embeddings index, auto-retrieves relevant code
OpenClaw/Nexu  : Config-driven — AGENTS.md declares what to load
Codex          : Repo-aware — scans structure, loads relevant files
Cline          : IDE-aware — open tabs + workspace files
```

**Трад-off:** Явный контроль (Aider) даёт точность, но требует усилий пользователя. Семантический поиск (Cursor) автоматичен, но может пропустить или включить неправильные файлы. Конфигурационный подход (OpenClaw) предсказуем, но требует предварительной настройки.

### Архитектура памяти

```python
# Type 1: No memory (Claude Code, Codex, Cursor, Cline)
# Each session starts fresh. Context comes from project files.
context = load_project_files()  # That's it

# Type 2: File-based memory (OpenClaw, Nexu)
# Persistent knowledge across sessions, user-editable.
context = (
    load_project_files()
    + load_memory("MEMORY.md")
    + load_daily_log(today)
    + load_wiki_if_relevant(task)
)

# Type 3: Embedded memory (proprietary platforms)
# Stored in vector DB, retrieved by similarity.
context = (
    load_project_files()
    + vector_search(task, memory_store)
)
```

### Расширяемость инструментов

| Подход | Проекты | Плюсы | Минусы |
|---|---|---|---|
| **Протокол MCP** | OpenClaw, Claude Code, Cline | Стандартный протокол, совместимость | Накладные расходы на настройку сервера |
| **Файлы навыков** | OpenClaw, Nexu | Автономные, делимые, маркетплейс | Свой формат |
| **Только встроенные** | Aider, Codex | Простота, предсказуемость | Ограниченная расширяемость |
| **Интеграция с IDE** | Cursor, Cline | Богатый контекст редактора | Привязка к IDE |

### Паттерны мультиагентности

Большинство харнесов — одиночные агенты. Поддержка мультиагентов существенно различается:

```python
# OpenClaw/Nexu: First-class sub-agent spawning
subagent = spawn(
    task="Research competitor pricing",
    model="gpt-4o",
    tools=["web_search", "web_fetch"],
)
# Result auto-announces when done — no polling needed

# Claude Code: Indirect multi-agent via exec
result = exec("claude-code --print 'Review this PR'")
# Works but no structured communication

# Others: No native multi-agent
# Workaround: multiple terminal sessions, manual coordination
```

| Минимальная настройка, просто работает | **Aider** |

## Чего не показывает таблица

- **Поддержка моделей** — Большинство харнесов агностичны к моделям (OpenClaw, Cline, Aider поддерживают многих провайдеров). Codex работает только с OpenAI. Claude Code ориентирован на Anthropic.
- **Стоимость** — Сильно варьируется в зависимости от выбора модели, размера контекста и сложности задач. Это не функция харнеса, а паттерн использования.
- **Скорость** — Зависит больше от латентности модели, чем от архитектуры харнеса. Поддержка стриминга универсальна.
- **Сообщество** — Open-source проекты (Aider, Claude Code, Cline, OpenClaw) имеют активные сообщества. Размер сообщества не равен качеству.

## Частые ошибки

- **Выбор на основе хайпа** — Выбирайте харнес, подходящий под Ваш рабочий процесс, а не тот, у которого больше всего звёзд на GitHub.
- **Игнорирование памяти** — Если Ваши задачи охватывают несколько сессий, харнес без постоянной памяти означает необходимость заново объяснять контекст каждый раз.
- **Полагать, что «open source» означает «бесплатно»** — Харнес бесплатен; API-вызовы модели — нет. Спланируйте бюджет на API.
