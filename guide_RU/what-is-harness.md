# Что такое харнес?

> **Ключевая идея:** Модели становятся товаром — GPT, Claude, Gemini сходятся по возможностям. Харнес — это настоящий конкурентный барьер: как Вы оркестрируете контекст, память, инструменты и жизненный цикл агента определяет, выпустите ли Вы чатбота или production-агента.

## Определение

**Харнес** — это обёртка среды выполнения, которая превращает голую языковую модель в **агента** — автономную систему, способную воспринимать среду, принимать решения и выполнять действия за несколько шагов для достижения целей.

Важно отличать здесь «агента» от более раннего использования. В 2023–2024 «агент» обычно означал *модель плюс инструменты* — Вы давали GPT инструмент веб-поиска и называли это агентом. Агенты, на которые нацелена инженерия харнесов, принципиально сложнее:

| Компонент | «Агент» 2023 | Агент эпохи харнесов |
|-----------|-------------|----------------------|
| Модель | ✅ LLM | ✅ LLM |
| Инструменты | ✅ Function calling | ✅ Динамическая система инструментов |
| Память | ❌ Stateless | ✅ Постоянная кросс-сессионная память |
| Управление контекстом | ❌ Наивное | ✅ Приоритетная сборка контекста |
| Оркестрация | ❌ Одноразовый ход | ✅ Агентный цикл с восстановлением ошибок |
| Среда выполнения | ❌ Процесс хоста | ✅ Песочничная среда |
| Защитные ограничения | ❌ Минимальные | ✅ Модель разрешений + границы доверия |

Харнес — это инженерный слой, обеспечивающий всё это. Без него у Вас чатбот, который может вызывать функции. С ним — агент, который может навигировать по кодовой базе, исправлять ошибки в нескольких файлах и коммитить результат — всё автономно.

## Структура харнеса

Каждый харнес, независимо от реализации, имеет четыре подсистемы:

```
┌──────────────────────────────────────────────┐
│                   HARNESS                     │
│                                               │
│  ┌──────────┐  ┌──────────┐  ┌────────────┐  │
│  │ Agentic  │  │   Tool   │  │  Memory &  │  │
│  │   Loop   │  │  System  │  │  Context   │  │
│  └──────────┘  └──────────┘  └────────────┘  │
│                                               │
│  ┌────────────────────────────────────────┐   │
│  │            Guardrails                  │   │
│  └────────────────────────────────────────┘   │
└──────────────────────────────────────────────┘
```

1. **Агентный цикл** — цикл «подумай → действуй → наблюдай», управляющий всем поведением агента. Модель рассуждает, вызывает инструмент, анализирует результат и повторяет, пока задача не будет выполнена.

2. **Система инструментов** — реестр возможностей, доступных агенту: файловый ввод-вывод, выполнение shell-команд, веб-поиск, вызовы API. Инструменты могут быть статическими (загружаются при запуске) или динамическими (загружаются по запросу через меню навыков).

3. **Память и контекст** — система, определяющая, что модель может *видеть*. Охватывает три различных аспекта:
   - **Контекст** — что попадает в текущий API-запрос (системный промпт, инструменты, файлы, история диалога)
   - **Память** — что сохраняется между сессиями (MEMORY.md, ежедневные логи, изучённые предпочтения)
   - **Сессия** — граница одного запуска агента (история сообщений, результаты инструментов, временное состояние)

4. **Защитные ограничения** — границы разрешений, обеспечение песочницы и ограничения безопасности. Что агент может и не может делать, и как предотвратить обход этих границ через prompt-injection.

Эти четыре подсистемы подробно рассматриваются в разделе [Основные концепции](/guide/agentic-loop).

## Минимальный пример

Простейший харнес — это цикл. Он неполон для продакшена, но структурно верен:

```python
import openai

client = openai.OpenAI()
tools = [{"type": "function", "function": {"name": "read_file", ...}}]

messages = [{"role": "system", "content": "You are a coding agent."}]
messages.append({"role": "user", "content": user_input})

# The agentic loop
while True:
    response = client.chat.completions.create(
        model="gpt-4o", messages=messages, tools=tools
    )
    msg = response.choices[0].message
    messages.append(msg)

    if not msg.tool_calls:
        print(msg.content)  # Done — model has no more actions
        break

    for call in msg.tool_calls:
        result = execute_tool(call.function.name, call.function.arguments)
        messages.append({
            "role": "tool",
            "tool_call_id": call.id,
            "content": result
        })
    # Loop back — model sees the tool results and decides next action
```

Каждый харнес — от 50-строчного скрипта до Claude Code — это вариация этого цикла. Сложность возникает из того, что Вы строите *вокруг* него: сборка контекста, персистентность памяти, оркестрация навыков, восстановление ошибок и песочница.

## Харнес vs. фреймворк vs. среда выполнения

Эти три термина часто путают. Это разные слои:

| Термин | Роль | Примеры |
|--------|------|---------|
| **Харнес** | Код оркестрации, оборачивающий модель в агента | Claude Code, Codex CLI, OpenClaw |
| **Фреймворк** | Библиотека, предоставляющая строительные блоки для построения харнесов | LangChain, CrewAI, AutoGen |
| **Среда выполнения (Runtime)** | Постоянный процесс, поддерживающий харнес, управляющий его жизненным циклом и подключающий к внешнему миру | OpenClaw runtime, Docker-контейнер, systemd-сервис |

Фреймворк помогает Вам *построить* харнес. Среда выполнения *хостит* харнес — поддерживает его жизнь, обрабатывает переподключения, планирует heartbeat'и и маршрутизирует сообщения. Сам харнес — это логика оркестрации: как собирается контекст, какие инструменты загружаются и как ведёт себя агентный цикл.

## Частые ошибки

- **Винят модель в проблемах харнеса** — Когда агент терпит неудачу, обычно это проблема контекста (загружены не те файлы, отсутствуют инструкции) или инструментов (некорректная схема, молчаливые ошибки), а не способность модели.
- **Проектирование с избытком с первого дня** — Начните с минимального цикла выше. Добавляйте память, когда нужна кросс-сессионная состоятельность. Добавляйте навыки, когда инструментов становится слишком много. Добавляйте защитные ограничения при переходе в продакшен.
- **Рассматривать контекстное окно как бесконечное** — Модель может рассуждать только о том, что находится в её контексте. Если критическая информация не собрана в промпте, она фактически не существует.
